Mathium Interpreter in C++

Description

I developed the Mathium interpreter, a programming language designed specifically for mathematical expressions and computations. This interpreter translates Mathium code into bytecode, which is then executed on a virtual machine. The project involved multiple phases: lexing, parsing into an abstract syntax tree (AST), semantic analysis, compilation into bytecode, and execution by the virtual machine.

Project Highlights

  • Phases of Interpretation
    • Lexing: Tokenizes the input code.
    • Parsing: Converts tokens into an Abstract Syntax Tree (AST).
    • Semantic Analysis: Ensures the code follows language rules and semantics.
    • Compilation: Translates AST into bytecode.
    • Execution: Executes bytecode on a virtual machine.
  • Language Features
    • Variable Declaration and Scope: Supports variable declaration, variable scopes, and variable shadowing.
    • Functionality: Includes function declaration and function overloading.
    • Operators Supported:
      • Addition (+): binary and unary
      • Subtraction (-): binary and unary
      • Multiplication (*): binary
      • Division (/): binary
      • Exponentiation (^): binary
      • Print (print): unary
    • Constant Declaration: Using the const keyword for immutable variables.
  • Future Enhancements
    • Optimization: Implementing optimizations to improve the interpreter's performance.
    • Standard Library: Adding a standard library with common mathematical functions.
    • Interactive Mode: Enabling an interactive mode for real-time code execution.
    • Custom Operator Declaration: Allowing users to define custom operators.

PersonalHub in C

Description

PersonalHub is a console-based personal electronic agenda developed as a team project during my second year in preparatory classes for our C programming class. The application, built using PDCurses for the user interface, includes features such as note-taking, task management with due dates, and a calendar, all accessible through a console interface.

Project Highlights

  • User Interface
    • Console-based UI: Developed entirely from scratch using PDCurses.
    • UI Elements: Created buttons, text fields, and text areas from scratch.
    • Navigation: Users can navigate through different "pages" such as login, registration, and their personal hub, using keyboard arrow keys.
    • Features
      • Login and Registration: Users can create an account and log in to access their personal hub.
      • Personal Hub: Each user has their own space to:
        • Notes: Create, read, edit, and delete notes
        • Tasks: Manage tasks with due dates.
        • Calendar: View a calendar with tasks and notes.
      • Data Storage & Security: Each user's data is stored in a separate database file, ensuring privacy and data integrity.
    • Technical Challenges and Solutions
      • UI Implementation: Overcame the challenge of creating a console-based UI with PDCurses, including handling user input and rendering UI elements.
        • Buttons: Printed text surrounded by spaces with background colors to simulate buttons.
        • Highlighting: Changed text and background colors to indicate the selected element.
        • Navigation: Used arrow keys to move between UI elements such as buttons and text fields.
        • Text Fields: Used cursor positioning and printed borders to create text fields.
        • Text Areas: Implemented scrolling text areas for notes and tasks.
      • Database Management: Implemented a custom database system to store user data securely and efficiently.
      • Task Management: Designed a task management system that allows users to add, edit, and delete tasks with due dates.
      • Calendar Integration: Integrated a calendar view that displays tasks and notes based on their due dates.
